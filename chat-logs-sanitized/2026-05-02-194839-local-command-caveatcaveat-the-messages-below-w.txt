     GitHub Actions 构建 Tauri 应用计划

     Context

     VidSum 项目目前只有一个 Gitleaks 密钥扫描的
     workflow（.github/workflows/gitleaks.yml），缺少 Tauri 应用的 CI
     构建流水线。用户需要在 GitHub Actions
     上实现三端（Windows/macOS/Linux）手动触发构建。

     主要挑战：
     - whisper-rs 依赖需要 cmake 和 C++ 编译器
     - Whisper 模型文件（77MB）未提交到 Git，需要在 CI 中下载
     - 各平台系统依赖不同（Ubuntu 需要 GTK/WebKit 等）

     实施方案

     新建文件：.github/workflows/build.yml

     创建一个 workflow 文件，核心结构如下：

     触发条件：仅 workflow_dispatch（手动触发）

     矩阵策略：三个平台并行构建
     - windows-latest → x86_64-pc-windows-msvc
     - macos-latest → aarch64-apple-darwin
     - ubuntu-22.04 → x86_64-unknown-linux-gnu

     构建步骤（按顺序）：

     1. 检出代码 — actions/checkout@v4
     2. 安装系统依赖
       - Ubuntu: cmake build-essential libgtk-3-dev libwebkit2gtk-4.1-dev 
     libappindicator3-dev librsvg2-dev patchelf
       - macOS: brew install cmake
       - Windows: choco install cmake + ilammy/msvc-dev-cmd@v1
     3. 安装 pnpm — pnpm/action-setup@v4（version: 9）
     4. 安装 Node.js — actions/setup-node@v4（node 20，启用 pnpm 缓存）
     5. 安装前端依赖 — pnpm install --frozen-lockfile
     6. 安装 Rust 工具链 — dtolnay/rust-toolchain@stable（带平台 target）
     7. 缓存 Rust 依赖 — Swatinem/rust-cache@v2（workspaces: src-tauri）
     8. 下载 Whisper 模型 — 用 actions/cache@v4 缓存，缓存未命中时从 HuggingFace
      下载 ggml-tiny.bin 到 src-tauri/models/
     9. 构建 Tauri 应用 — tauri-apps/tauri-action@v0（tauriScript: "pnpm
     tauri"）
     10. 上传构建产物 — actions/upload-artifact@v4，按平台上传
     msi/exe/dmg/AppImage/deb 等

     关键设计决策


     ┌─────────────┬────────────────────────────┬───────────────────────────────
     ┐
     │    决策     │            选择            │             理由
     │
     ├─────────────┼────────────────────────────┼───────────────────────────────
     ┤
     │ 构建 action │ tauri-apps/tauri-action@v0 │ 自动处理产物路径，社区标准
     │
     ├─────────────┼────────────────────────────┼───────────────────────────────
     ┤
     │ pnpm        │ tauriScript: "pnpm tauri"  │ 项目强制使用 pnpm
     │
     │ 执行方式    │                            │
     │
     ├─────────────┼────────────────────────────┼───────────────────────────────
     ┤
     │ 模型下载    │ actions/cache + curl       │ 77MB
     │
     │             │                            │ 文件适合缓存，避免重复下载
     │
     ├─────────────┼────────────────────────────┼───────────────────────────────
     ┤
     │ Ubuntu 版本 │ ubuntu-22.04               │ WebKit 依赖更稳定
     │
     ├─────────────┼────────────────────────────┼───────────────────────────────
     ┤
     │ macOS       │ aarch64-apple-darwin       │ macos-latest 当前为 ARM64
     │
     │ target      │                            │
     │
     ├─────────────┼────────────────────────────┼───────────────────────────────
     ┤
     │ 构建超时    │ 45 分钟                    │ whisper.cpp 首次编译较慢
     │
     └─────────────┴────────────────────────────┴───────────────────────────────
     ┘

     签名（可选）

     workflow 中预置了 TAURI_SIGNING_PRIVATE_KEY 和
     TAURI_SIGNING_PRIVATE_KEY_PASSWORD 环境变量。如未配置 secret，Tauri
     会构建未签名应用，不影响功能。

     涉及文件

     - 新建: .github/workflows/build.yml
     - 只读参考: src-tauri/tauri.conf.json、src-tauri/Cargo.toml、package.json

     验证方式

     1. 提交 workflow 文件并推送到 GitHub
     2. 在仓库 Actions 页面手动触发 "构建发布" workflow
     3. 确认三个平台的 job 都能成功完成
     4. 下载 artifact 验证各平台安装包可正常运行
